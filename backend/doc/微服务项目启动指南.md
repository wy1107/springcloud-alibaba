# Spring Cloud Alibaba 微服务项目启动指南

> **版本基线**：Spring Boot 2.6.13 + Spring Cloud 2021.0.5 + Spring Cloud Alibaba 2021.0.5.0 + MySQL 5.7 + JDK 1.8

---

## 1. 环境准备清单

### 1.1 必备软件

| 软件 | 最低版本 | 推荐版本 | 验证命令 | 备注 |
|------|---------|---------|---------|------|
| JDK | 1.8.0+ | 1.8.0_202 | `java -version` | 必须为JDK而非JRE |
| Maven | 3.6.3+ | 3.8.6 | `mvn -version` | 确保`JAVA_HOME`已配置 |
| MySQL | 5.7 | 5.7.44 | `mysql -V` | 使用InnoDB引擎 |
| Git | 2.0+ | 最新LTS | `git --version` | 可选 |

### 1.2 环境变量检查

```bash
# 必须
echo %JAVA_HOME%        # 应指向JDK安装目录，如 C:\Program Files\Java\jdk1.8.0_202
echo %MAVEN_HOME%       # 应指向Maven安装目录，如 D:\develop_tools\apache-maven-3.8.6
echo %PATH%             # 应包含 %JAVA_HOME%\bin 和 %MAVEN_HOME%\bin

# 可选（Docker部署时）
docker --version        # Docker 20.10+
docker-compose --version # docker-compose 1.29+
```

### 1.3 Maven配置建议

确认 `settings.xml` 中配置了国内镜像（加速依赖下载）：

```xml
<mirror>
    <id>aliyun</id>
    <mirrorOf>central</mirrorOf>
    <url>https://maven.aliyun.com/repository/central</url>
</mirror>
```

> ⚠️ **生产注意**：生产环境应使用私有Maven仓库（如Nexus），确保依赖版本可控、不可篡改。

## 2. 服务依赖关系与启动顺序

### 2.1 依赖关系图

```
                    ┌──────────────┐
                    │    MySQL     │
                    │  (shop DB)   │
                    └──────┬───────┘
                           │
              ┌────────────┼────────────┐
              │            │            │
              ▼            ▼            ▼
      ┌──────────────┐ ┌──────────────┐ ┌──────────────┐
      │ shop-product │ │  shop-user   │ │  shop-order  │
      │  端口: 8081  │ │  端口: 8071  │ │  端口: 8091  │
      └──────┬───────┘ └──────────────┘ └──────┬───────┘
             │                                │
             │    RestTemplate HTTP调用        │
             ◄────────────────────────────────┘
```

### 2.2 启动顺序

```
步骤1: MySQL           → 必须最先启动，所有微服务依赖数据库
步骤2: shop-common     → mvn install 安装到本地仓库（非运行时服务）
步骤3: shop-product    → 商品服务（订单服务调用它，必须先启动）
步骤4: shop-user       → 用户服务（无前置依赖，可与product并行启动）
步骤5: shop-order      → 订单服务（依赖product的HTTP接口，最后启动）
```

> ⚠️ **关键**：`shop-order` 通过 `RestTemplate` 硬编码调用 `http://localhost:8081/product/{pid}`，如果 `shop-product` 未启动，下单接口将返回 500 错误。

### 2.3 各服务信息总览

| 模块 | 端口 | 服务名 | 启动类 | 角色 |
|------|------|--------|--------|------|
| shop-common | - | - | -（公共模块） | 被其他模块依赖 |
| shop-user | 8071 | service-user | `UserApplication` | 服务提供者 |
| shop-product | 8081 | service-product | `ProductApplication` | 服务提供者 |
| shop-order | 8091 | service-order | `OrderApplication` | 服务消费者 |

## 3. 配置参数详解

### 3.1 配置文件加载顺序（优先级由高到低）

```
bootstrap.properties → bootstrap.yml → application.properties → application.yml
```

> 当前项目仅使用 `application.yml`。后续集成 Nacos Config 时需新增 `bootstrap.yml`，配置优先级高于 `application.yml`。

### 3.2 shop-product 配置模板（端口 8081）

```yaml
server:
  port: 8081                          # 服务端口号

spring:
  application:
    name: service-product             # 服务名，后续注册到Nacos时使用
  datasource:
    driver-class-name: com.mysql.jdbc.Driver    # MySQL 5.x 驱动类
    url: jdbc:mysql://localhost:3306/shop?useUnicode=true&characterEncoding=utf-8&useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true
    username: root                    # ⚠️ 生产环境禁止使用root
    password: onetwo12                # ⚠️ 生产环境必须加密，见下方说明
  jpa:
    hibernate:
      ddl-auto: update               # 自动根据实体更新表结构
    show-sql: true                    # 打印SQL语句（生产环境设为false）
    database-platform: org.hibernate.dialect.MySQL5InnoDBDialect  # MySQL 5.7 InnoDB方言
```

### 3.3 shop-user 配置模板（端口 8071）

```yaml
server:
  port: 8071

spring:
  application:
    name: service-user
  datasource:
    driver-class-name: com.mysql.jdbc.Driver
    url: jdbc:mysql://localhost:3306/shop?useUnicode=true&characterEncoding=utf-8&useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true
    username: root
    password: onetwo12
  jpa:
    hibernate:
      ddl-auto: update
    show-sql: true
    database-platform: org.hibernate.dialect.MySQL5InnoDBDialect
```

### 3.4 shop-order 配置模板（端口 8091）

```yaml
server:
  port: 8091

spring:
  application:
    name: service-order
  datasource:
    driver-class-name: com.mysql.jdbc.Driver
    url: jdbc:mysql://localhost:3306/shop?useUnicode=true&characterEncoding=utf-8&useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true
    username: root
    password: onetwo12
  jpa:
    hibernate:
      ddl-auto: update
    show-sql: true
    database-platform: org.hibernate.dialect.MySQL5InnoDBDialect
```

### 3.5 关键配置参数说明

| 参数 | 说明 | 开发环境 | 生产环境 |
|------|------|---------|---------|
| `server.port` | 服务监听端口 | 按上表分配 | 通过启动参数覆盖：`--server.port=xxx` |
| `spring.datasource.driver-class-name` | MySQL 5.x 使用 `com.mysql.jdbc.Driver`；MySQL 8.x 使用 `com.mysql.cj.jdbc.Driver` | 按版本选择 | 必须与MySQL版本匹配 |
| `spring.datasource.url` | `useSSL=false` 禁用SSL；`serverTimezone=Asia/Shanghai` 设置时区 | 如上 | 生产必须 `useSSL=true` |
| `spring.jpa.hibernate.ddl-auto` | `update` 自动建表；`none` 不操作；`validate` 仅校验 | `update` | **必须 `none` 或 `validate`**，禁止自动DDL |
| `spring.jpa.show-sql` | 控制台打印SQL | `true` | **`false`**，避免性能损耗 |
| `spring.jpa.database-platform` | Hibernate方言 | `MySQL5InnoDBDialect` | MySQL 8.x 使用 `MySQL8Dialect` |

> ⚠️ **生产环境安全要求**：
> - 数据库密码禁止明文，使用 Jasypt 加密：`password: ENC(加密内容)`
> - 数据库账号禁止使用 `root`，应创建最小权限账号
> - `ddl-auto` 必须设为 `none`，使用 Flyway/Liquibase 管理数据库版本

## 4. 健康检查与启动验证

### 4.1 数据库初始化

```sql
-- 创建数据库
CREATE DATABASE IF NOT EXISTS shop DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

USE shop;

-- JPA的 ddl-auto=update 会自动建表，以下仅需插入测试数据
INSERT INTO user (username, password, telephone) VALUES
('张三', '123456', '13800138001'),
('李四', '123456', '13800138002');

INSERT INTO product (pname, pprice, stock) VALUES
('小米手机', 1999.00, 100),
('华为笔记本', 5999.00, 50),
('苹果平板', 3999.00, 80);
```

### 4.2 构建项目

```bash
# Step 1: 安装公共模块到本地Maven仓库（必须先执行）
mvn clean install -DskipTests -f shop-common/pom.xml

# Step 2: 打包各微服务
mvn clean package -DskipTests -f shop-product/pom.xml
mvn clean package -DskipTests -f shop-user/pom.xml
mvn clean package -DskipTests -f shop-order/pom.xml
```

### 4.3 启动服务

**方式一：命令行启动**

```bash
# 按顺序启动，每个服务等待启动完成再启动下一个
java -jar shop-product/target/shop-product-1.0-SNAPSHOT.jar
java -jar shop-user/target/shop-user-1.0-SNAPSHOT.jar
java -jar shop-order/target/shop-order-1.0-SNAPSHOT.jar
```

**方式二：IDE启动（推荐开发时使用）**

在IDEA中分别运行各模块的启动类：
- `com.example.shop.product.ProductApplication`
- `com.example.shop.user.UserApplication`
- `com.example.shop.order.OrderApplication`

**方式三：自定义JVM参数启动**

```bash
# 指定端口和内存参数
java -Xms256m -Xmx512m -Dserver.port=8081 -jar shop-product/target/shop-product-1.0-SNAPSHOT.jar
```

### 4.4 启动验证步骤

| 序号 | 验证项 | 操作 | 预期结果 |
|------|--------|------|---------|
| 1 | MySQL连通性 | `mysql -uroot -p -e "USE shop; SHOW TABLES;"` | 显示 `product`、`user`、`shop_order` 表 |
| 2 | 商品服务 | `curl http://localhost:8081/product/1` | 返回 `{"pid":1,"pname":"小米手机","pprice":1999.0,"stock":100}` |
| 3 | 用户服务 | `curl http://localhost:8071/user/1` | 返回 `{"uid":1,"username":"张三",...}` |
| 4 | 下单接口 | `curl "http://localhost:8091/order/create?pid=1&uid=1"` | 返回订单JSON，包含商品名和价格 |
| 5 | 端口监听 | `netstat -ano | findstr "8071 8081 8091"` | 三个端口均为 LISTENING 状态 |

## 5. 常见启动问题排查手册

### 5.1 编译构建阶段

| # | 错误现象 | 原因分析 | 解决方法 |
|---|---------|---------|---------|
| 1 | `Unable to find main class` | 父pom的 `spring-boot-maven-plugin` 对 `shop-common`（无main方法）执行repackage | 父pom中插件设置 `<skip>true</skip>`，各微服务子模块中设置 `<skip>false</skip>` |
| 2 | `程序包com.example.shop.common.entity不存在` | `shop-common` 未安装到本地Maven仓库 | 先执行 `mvn install -DskipTests -f shop-common/pom.xml` |
| 3 | `lombok.extern.slf4j不存在` | Lombok 在 `shop-common` 中为 `optional`，不会传递依赖 | 在使用 `@Slf4j` 的子模块pom中显式添加 lombok 依赖 |
| 4 | `Failed to delete target/*.jar` | Maven clean 时jar被运行中的Java进程锁定 | 先 `taskkill /F /PID <pid>` 停止微服务，再重新打包 |
| 5 | `Non-resolvable parent POM` | 父工程未先安装到本地仓库 | 在项目根目录先执行 `mvn install -DskipTests -N`（仅安装父pom） |

### 5.2 数据库连接阶段

| # | 错误现象 | 原因分析 | 解决方法 |
|---|---------|---------|---------|
| 6 | `Access denied for user 'root'` | MySQL密码不正确 | 修改 `application.yml` 中的 `spring.datasource.password` |
| 7 | `Communications link failure` | MySQL未启动或端口不对 | 检查MySQL服务状态：`net start | findstr MySQL` |
| 8 | `Access to DialectResolutionInfo cannot be null when 'hibernate.dialect' not set` | Hibernate无法自动识别MySQL方言（通常因连接失败导致） | yml中显式添加 `database-platform: org.hibernate.dialect.MySQL5InnoDBDialect` |
| 9 | `Loading class 'com.mysql.jdbc.Driver'. This is deprecated.` | MySQL Connector/J 5.1 中旧驱动类已弃用提示 | 不影响运行；若需消除警告可升级到MySQL 8.x并使用 `com.mysql.cj.jdbc.Driver` |

### 5.3 运行时阶段

| # | 错误现象 | 原因分析 | 解决方法 |
|---|---------|---------|---------|
| 10 | `Port 8081 was already in use` | 端口被其他进程占用 | 查找并终止：`netstat -ano | findstr :8081` → `taskkill /F /PID <pid>`，或修改 `server.port` |
| 11 | `MySQLSyntaxErrorException: near 'order'` | `order` 是MySQL保留关键字，JPA默认以类名作表名 | 实体类添加 `@Table(name = "shop_order")` 避免冲突 |
| 12 | 下单接口返回500 | `shop-product` 未启动，`RestTemplate` 调用失败 | 确保 `shop-product` 在 `shop-order` 之前启动 |
| 13 | `spring.jpa.open-in-view is enabled by default` | OpenEntityManagerInViewFilter 默认开启的警告 | 添加配置 `spring.jpa.open-in-view: false` |
| 14 | `HikariPool-1 - Exception during pool initialization` | 连接池初始化失败，通常是数据库不可达 | 按序排查：MySQL是否启动 → 账号密码 → 网络连通性 |

### 5.4 排查流程图

```
启动失败
  │
  ├─ 编译报错？
  │    ├─ "程序包xxx不存在" → shop-common未install → mvn install shop-common
  │    ├─ "Unable to find main class" → spring-boot-maven-plugin配置 → 父pom skip=true
  │    └─ "lombok不存在" → 子模块缺少lombok依赖 → 添加lombok依赖
  │
  ├─ 数据库连接失败？
  │    ├─ "Access denied" → 密码错误 → 检查application.yml
  │    ├─ "Communications link failure" → MySQL未启动 → 启动MySQL
  │    └─ "DialectResolutionInfo cannot be null" → 方言未设置 → 添加database-platform
  │
  ├─ 端口冲突？
  │    └─ "Port xxx already in use" → netstat查端口 → taskkill或改端口
  │
  └─ 运行时500？
       ├─ RestTemplate调用失败 → 被调服务未启动 → 先启动product再启动order
       └─ SQL语法错误 → 表名为保留字 → 添加@Table注解指定表名
```


### 6.2 异常日志标记

| 日志关键字 | 含义 | 处理方式 |
|-----------|------|---------|
| `APPLICATION FAILED TO START` | 启动失败总标记 | 查看下方具体原因 |
| `Exception during pool initialization` | 数据库连接失败 | 检查MySQL状态和连接参数 |
| `Port 8081 was already in use` | 端口冲突 | 终止占用进程或换端口 |
| `Error executing DDL` | JPA建表失败 | 通常是表名保留字冲突 |
| `GenerationTarget encountered exception` | DDL执行异常 | 检查SQL语法和表名 |
| `Communications link failure` | 网络连接失败 | 检查MySQL是否可达 |

### 6.3 日志级别调整

如需更详细的日志输出，在 `application.yml` 中添加：

```yaml
logging:
  level:
    root: INFO
    com.example.shop: DEBUG              # 项目代码DEBUG级别
    org.hibernate.SQL: DEBUG             # SQL语句
    org.hibernate.type.descriptor.sql.BasicBinder: TRACE  # SQL参数绑定
    org.springframework.web: DEBUG       # Spring MVC请求日志
```

### 6.4 启动耗时参考

| 服务 | 正常启动时间 | 异常阈值 |
|------|------------|---------|
| shop-product | 3~6秒 | > 15秒需排查 |
| shop-user | 3~6秒 | > 15秒需排查 |
| shop-order | 5~10秒 | > 20秒需排查 |

> 启动耗时主要受网络环境（首次下载依赖）和数据库连接速度影响。首次启动因JPA建表会略慢。

## 附录：快速启动脚本

### Windows (start-all.bat)

```bat
@echo off
echo [1/3] Starting shop-product...
start "shop-product" java -jar shop-product/target/shop-product-1.0-SNAPSHOT.jar
timeout /t 10 /nobreak >nul

echo [2/3] Starting shop-user...
start "shop-user" java -jar shop-user/target/shop-user-1.0-SNAPSHOT.jar
timeout /t 8 /nobreak >nul

echo [3/3] Starting shop-order...
start "shop-order" java -jar shop-order/target/shop-order-1.0-SNAPSHOT.jar

echo All services started. Verifying...
timeout /t 10 /nobreak >nul
curl -s http://localhost:8081/product/1
curl -s http://localhost:8071/user/1
curl -s "http://localhost:8091/order/create?pid=1&uid=1"
pause
```

### 停止全部服务 (stop-all.bat)

```bat
@echo off
for /f "tokens=5" %%a in ('netstat -ano ^| findstr ":8081 .*LISTENING"') do taskkill /F /PID %%a 2>nul
for /f "tokens=5" %%a in ('netstat -ano ^| findstr ":8071 .*LISTENING"') do taskkill /F /PID %%a 2>nul
for /f "tokens=5" %%a in ('netstat -ano ^| findstr ":8091 .*LISTENING"') do taskkill /F /PID %%a 2>nul
echo All services stopped.
```
